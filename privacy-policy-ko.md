# GymCheck 개인정보처리방침

**시행일자**: 2026년 5월 4일
**최종 개정일**: 2026년 5월 4일

GymCheck(이하 "서비스")는 이용자의 개인정보를 소중히 여기며, 「개인정보 보호법」 및 관련 법령을 준수하기 위해 다음과 같이 개인정보처리방침을 수립·공개합니다.

---

## 1. 개인정보의 수집 항목 및 수집 방법

### 1.1 회원가입 시 수집 항목
| 항목 | 수집 방법 | 필수/선택 |
|---|---|---|
| 이메일 주소 | 이메일 회원가입 시 직접 입력 | 필수 |
| 비밀번호 (암호화 저장) | 이메일 회원가입 시 직접 입력 | 필수 |
| 이름, 이메일 (Apple 제공) | Apple 로그인 사용 시 자동 수집 | 선택 |

### 1.2 서비스 이용 시 자동 수집 항목
- 운동 기록 (날짜, 운동 부위)
- 인바디 측정 결과 (체중, 골격근량, 체지방률 등 InBody 결과지에 포함된 측정 항목)
- 인바디 결과지 사진 (이용자가 직접 업로드한 경우, 단말기 내에만 저장)
- 사용자가 설정한 프로필 정보 (이름, 운동 시작일, 선호 언어)

### 1.3 단말기 권한
| 권한 | 사용 목적 | 거부 시 영향 |
|---|---|---|
| 카메라 | 인바디 결과지 촬영 | 사진 촬영 기능만 사용 불가, 다른 기능 정상 사용 |
| 사진 라이브러리 (읽기) | 인바디 결과지 사진 불러오기 | 사진 불러오기 기능만 사용 불가 |
| 사진 라이브러리 (쓰기) | 운동 기록 카드를 사진으로 저장 | 카드 저장 기능만 사용 불가 |

---

## 2. 개인정보의 이용 목적

수집한 개인정보는 다음 목적으로만 사용됩니다.
1. 회원 식별 및 인증
2. 운동·체성분 기록의 저장 및 조회
3. 여러 기기 간 데이터 동기화
4. 통계 분석 화면 제공
5. 서비스 운영상 필요한 공지 전달

---

## 3. 개인정보의 보유 및 이용 기간

- 회원 가입 시점부터 **회원 탈퇴 시점까지** 보유합니다.
- 회원이 앱 내에서 **계정 삭제** 기능을 실행하면 다음 데이터가 즉시 삭제됩니다:
  - Firebase Authentication 의 계정 정보 (이메일, 인증 토큰)
  - Cloud Firestore 에 저장된 모든 운동 기록, 인바디 기록
  - 단말기에 저장된 로컬 데이터 (앱 삭제 시)
- 관계 법령에 따라 보존이 필요한 경우 해당 기간 동안만 별도 분리 보관합니다.

---

## 4. 개인정보의 제3자 제공

서비스는 이용자의 개인정보를 외부에 제공하지 않습니다. 다만 아래의 경우는 예외입니다.
1. 이용자가 사전에 동의한 경우
2. 법령에 의거한 수사기관 등의 요청이 있는 경우

---

## 5. 개인정보 처리 위탁

원활한 서비스 제공을 위해 다음 업체에 개인정보 처리를 위탁하고 있습니다.

| 수탁업체 | 위탁 업무 | 위탁 항목 |
|---|---|---|
| Google LLC (Firebase) | 인증, 데이터베이스(Cloud Firestore), 서버 운영 | 이메일, 인증 정보, 운동 기록, 인바디 기록 |
| Google LLC (AdMob) | 광고 노출 및 측정 | 광고 식별자, 대략적 위치(IP), 기기 정보 |
| Google LLC (Vertex AI in Firebase / Gemini) | 인바디 결과지 사진 분석 | 이용자가 업로드한 결과지 사진(일회성, 저장 안 됨) |
| Apple Inc. (StoreKit) | 인앱 결제 처리 (광고 제거 구독) | 결제 정보는 Apple 이 직접 처리, 운영자는 결제 토큰만 수신 |
| Apple Inc. | Sign in with Apple 인증 | 이메일, 이름 (이용자가 Apple 로그인 사용 시) |

> 인바디 결과지 사진은 측정 항목 추출 직후 폐기되며 저장·재사용되지 않습니다.

위탁받은 업체는 「개인정보 보호법」 및 각국의 개인정보 보호 관련 법령을 준수합니다.

- **이전받는 자의 정보**: Google LLC ([https://policies.google.com/privacy](https://policies.google.com/privacy))

## 10. 안전성 확보 조치

서비스는 다음 기술적·관리적 조치를 통해 개인정보의 안전성을 확보합니다.
- 모든 통신 구간 HTTPS/TLS 암호화
- Firebase 보안 규칙을 통한 사용자별 데이터 격리 (본인 데이터만 접근 가능)
- 비밀번호는 단방향 해시(bcrypt) 처리되어 서버에도 평문으로 저장되지 않음
- 정기적인 보안 점검 및 취약점 패치
